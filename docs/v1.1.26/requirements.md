## 二、需求明细

### R1. 主题切换改为图标切换（非开关按钮）

**现状：** 右上角 Header 使用一个滑块开关（toggle switch）+ 左右各一个 Moon/Sun 静态图标做主题切换。

**目标：**
- 移除滑块开关按钮
- 改为单个可点击图标：当前亮色模式显示月亮图标（点击切换到暗色），当前暗色模式显示太阳图标（点击切换到亮色）
- 图标风格：苹果系统质感，线性（outline）风格，与 lucide-vue-next 现有图标一致
- 点击即切换，无中间状态

### R2. 语言切换按钮

**目标：**
- 在主题切换图标**左侧**新增语言切换图标按钮
- 当前支持中/英切换，点击切换语言
- 图标风格：线性图标，与主题切换图标风格一致
- 切换后全局文案即时生效（无需刷新）
- **国际化框架要求：** 提前搭好 i18n 基础架构（vue-i18n），支持后续扩展多国语言
  - 语言文件放 `src/client/locales/` 目录，按语言代码分文件（`zh.ts`、`en.ts`）
  - 所有现有硬编码中文文案迁移到 i18n key 引用
  - 语言偏好持久化到 localStorage

### R3. 帮助按钮 + 帮助页面

**目标：**
- 在语言切换图标和主题切换图标**之后**（即最右侧）新增帮助图标按钮
- 图标使用线性 info 标记图标（如 lucide `Info`）
- 点击跳转到帮助页面（路由 `/help`）
- 帮助页面内容：
  - **飞书机器人板块**（已实现内容）：创建企业内部应用 → 添加机器人能力 → 配置权限 → 获取 App ID/Secret → 绑定到 Bridge 的完整操作指引
  - **其他平台板块**（Telegram / QQ / 微信）：占位卡片，点击显示「待实现」提示
- 帮助页面风格与现有页面一致（glass-card 卡片式布局）

### R4. 面包屑导航

**目标：**
- 全局新增面包屑组件，位于页面标题上方或内容区顶部
- 面包屑根据当前路由自动生成，支持一级菜单分组 → 子页面的层级展示
- 路由层级映射规则：
  - 首页：`首页`
  - 机器管理：`运维中心 / 机器`
  - Bots 管理：`绑定管理 / Bots`
  - 服务商：`绑定管理 / 服务商`
  - 绑定：`绑定管理 / 绑定`
  - 实时日志：`日志 / 实时日志`
  - 审计日志：`日志 / 审计日志`
  - 扣费日志：`日志 / 扣费日志`
  - 设置：`设置`
  - 帮助：`帮助`
- 点击面包屑任意层级可跳转对应页面
- 样式：苹果风格，当前页不高亮（普通文字），上级层级可点击（带链接色），分隔符用 `/` 或 `›`
- 面包屑组件放 `src/client/components/Breadcrumb.vue`

### R5. 版本升级

- package.json version 升级为 1.1.26
- maintain.md 和 plan.md 同步更新

### R6. 侧边栏菜单展开 bug 修复

**现状：** 进入首页后，所有一级分组菜单（运维中心、绑定管理、日志）都自动展开。

**目标：**
- 初始状态只展开当前路由所在的分组，其余收起
- 切换路由时，自动展开目标分组，收起其他分组（可选，但至少初始状态不能全展开）
- 修改 `expandedGroups` 默认值逻辑：不再硬编码所有分组为展开

## 三、技术约束

1. 前端技术栈：Vue 3 + TypeScript + Vite + Tailwind CSS
2. 图标库：lucide-vue-next（已有）
3. i18n 库：vue-i18n（需新增）
4. 样式风格：黑白灰苹果质感，线性图标
5. 所有文案禁止硬编码，走 i18n key

---

## 四、验收标准

1. 主题切换：点击图标即切换，无滑块开关
2. 语言切换：点击图标切换中/英，所有页面文案即时切换
3. 帮助页面：飞书板块内容完整可读，其他平台占位显示「待实现」
4. 面包屑：所有页面顶部显示正确层级，点击可跳转
5. 侧边栏：只有当前路由所在分组展开
6. i18n：新增中文文案全部走 key，无硬编码
