# v1.1.25 需求文档

> 版本：v1.1.25
> 日期：2026-05-17
> 状态：需求确认

---

## 一、需求背景

v1.1.24 修复了决策面板不转发的核心 Bug（⏵⏵ 内联格式识别），用户已可通过飞书文本回复与 cc 决策面板交互。

当前痛点：
1. **cc 工作时用户只看到"处理中"**，不知道 cc 在做什么（读文件？执行命令？等待网络？），体验焦虑
2. **回复卡片缺少统计信息**，不知道 cc 花了多少时间、调了哪些工具
3. **长回复占满聊天窗口**，代码块/工具输出很长时刷屏严重
4. **缺少快捷操控手段**，想中断执行、切换模型、查看状态都要手动进入 Web 面板

---

## 二、需求清单

### 需求 1：工具调用实时状态卡片

**用户故事**：当 cc 正在工作时，我希望看到 cc 当前在执行什么操作（如"正在执行 Bash 命令"或"正在读取文件 auth.py"），而不是只看到"处理中"。

**验收标准**：
- [ ] 处理中卡片显示最近 1-3 个工具调用名称（如 `Bash(git push)`、`Read(auth.py)`）
- [ ] 精简显示工具名（最多 30 字符），超长截断
- [ ] 卡片保留耗时信息
- [ ] 卡片底部有「中断」按钮（点击发送 Escape 到 cc）

**不做**：
- 不显示工具调用的输出内容（太长）
- 不做工具调用的详细参数展示

---

### 需求 2：耗时与工具统计

**用户故事**：收到 cc 回复时，我想知道 cc 花了多长时间、调用了几次工具，方便评估成本。

**验收标准**：
- [ ] 回复卡片底部 note 显示：`⏱ 思考 Ns · 执行 Ns · 总计 Ns | 🔧 Bash×2 Edit×1`
- [ ] 从 capturePane 中提取 `✻ Brewed/Cooked/Sautéed for Xs` 耗时行
- [ ] 统计本轮回复中的工具调用次数（按类型分组）
- [ ] 耗时提取不到时不显示耗时部分，只显示工具统计

**不做**：
- 不做 token/费用精确统计（tmux 抓屏拿不到）
- 不做累计跨会话统计

---

### 需求 3：可折叠长回复

**用户故事**：cc 有时候返回很长的代码或工具输出，我希望核心结论直接可见，长代码/详细输出可以折叠展开。

**验收标准**：
- [ ] 回复内容 > 1500 字符时，自动将后半部分放入 `collapsible_panel` 折叠
- [ ] 折叠面板标题显示「代码详情（N 行）」或「详细输出（N 行）」
- [ ] 前 1500 字符保留在主区域（核心结论/摘要一般在前部）
- [ ] 回复 ≤ 1500 字符时不变，保持现有全量展示

**不做**：
- 不做智能摘要（只做字符数阈值切分）
- 不做多层折叠

---

### 需求 4：/命令快捷交互

**用户故事**：我希望在飞书直接输入 `/status` 查看进程状态，或 `/interrupt` 中断执行，不用打开 Web 面板。

**验收标准**：
- [ ] 消息以 `/` 开头时不转发给 cc，由 bridge 本地处理
- [ ] 支持以下命令：

| 命令 | 功能 | 输出 |
|---|---|---|
| `/status` | 查看所有绑定进程实时状态 | 状态看板卡片（table: 进程/模型/状态/耗时） |
| `/interrupt` | 中断当前 cc 执行 | 发 Escape + 确认卡片 |
| `/model <id>` | 切换模型（重启 cc 会话） | 确认卡片 + 重启 |
| `/effort <level>` | 调整 effort（重启 cc 会话） | 确认卡片 + 重启 |

- [ ] 未知 `/` 命令返回帮助卡片
- [ ] `/` 命令不区分大小写

**不做**：
- 不做 `/history`（需要持久化，成本高）
- 不做 `/cost`（tmux 拿不到 token 数据）
- 不做 `/permissions`（权限模式切换涉及安全，后续版本）
- 不影响正常消息转发（非 `/` 开头的消息照常发给 cc）

## 三、优先级与依赖

|---|---|---|---|
| 1. 工具调用状态卡片 | P0 | extractToolCalls() | 第 1 个做 |

## 四、非功能需求

- 不影响现有消息转发、决策面板、卡片回调等已实现功能
- 所有新增卡片样式与现有卡片风格统一（黑白灰主题）
- /命令响应延迟 < 2s
