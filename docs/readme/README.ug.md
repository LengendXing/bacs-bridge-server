## 🌟 پروژېكتىغا قىسقىچە تونۇشتۇرۇش

**feishu-claudecode-bridge** بولسا 飞书 بوتى بىلەن AI CLI قوراللىرى（Claude Code / Codex）نى قوش يۆنىلىشلىك كۆۋرۈكلەشتۈرىدىغان سىستېما. ئۇ دائىملىق Bridge Server ئارقىلىق 飞书 ئۇچۇر ۋەقەلىرىنى بەلگىلەنگەن CLI جەريانىغا يۆتكەيدۇ، ئاندىن CLI نىڭ جاۋابىنى 飞书 توپى ياكى 飞书 خۇسۇسىي پاراڭغا قايتۇرۇپ ئەۋەتىدۇ.

قوللىنىش كۆرۈنۈشلىرى：
- كوللېكتىپتا 飞书 توپى ئارقىلىق AI پروگرامما ئىشلەش ۋەزىپىلىرىنى ھەمكارلىشىپ قوزغىتىش
- PC بولمىغان ئەھۋالدا（يانفون، تاختا كومپيۇتېر）، مۇلازىمېتىردىكى Claude Code نى يىراقتىن كونترول قىلىش
- ئوخشاش بىر مۇلازىمېتىردىكى كۆپلىگەن AI جەريانلىرىنى ئوخشاش بولمىغان 飞书 توپلىرىغا باغلاپ، پروژېكت ئايرىش
- بروۋزېر ياكى ئاندروئىد تېرمىنالى ئارقىلىق AI بىر تەرەپ قىلىش سۈرئىتى، خاتىرە ۋە سۆھبەت Timeline نى ھەقىقىي ۋاقىتتا كۆزىتىش

## ✨ يادرولۇق ئىقتىدارلار

| مودۇل | ئىقتىدار |
|------|------|
| **كۆپ ماشىنا باغلاش** | ئوخشاش مۇلازىمېتىردا بىرلا ۋاقىتتا كۆپلىگەن CLI جەريانىنى（cc-a / cc-b / codex-x ...）باشقۇرۇشنى قوللايدۇ، ھەربىر جەريان مۇستەقىل ھالدا بىر 飞书 بوتىغا باغلىنىدۇ |
| **يىراق ماشىنا باشقۇرۇش** | ئىچكى SSH Executor بىلەن يەرلىك + كۆپ يىراق ماشىنىنىڭ tmux session نى بىر تۇتۇپ باشقۇرىدۇ |
| **قوش CLI قوللاش** | Claude Code（cc）ۋە Codex ئىككى خىل CLI ماسلاشتۇرغۇچىسى، شۇنىڭ بىلەن بىر ۋاقىتتا ئارىلاشتۇرۇپ ئىشلىتىشكە بولىدۇ |
| **مۇلازىمەت بىلەن تەمىنلىگۈچىنى ئەركىن تەڭشەش** | ئۆز ئىچىگە ئالغان Anthropic / OpenAI قاتارلىق مۇلازىمەت بىلەن تەمىنلىگۈچى تەڭشىكى، ئۆزى بەلگىلەنگەن base_url ۋە API Key نى قوللايدۇ |
| **كۆپ مودېل + Effort** | cc بولسا low~max نى قوللايدۇ، codex بولسا minimal~xhigh نى قوللايدۇ، مودېل maxEffort بويىچە ئاپتوماتىك كېسىلىدۇ |
| **Web Terminal** | بروۋزېر ئىچىدىكى xterm بىۋاسىتە tmux pane غا ئۇلىنىدۇ، كۆرگەنلا نەرسە ئېرىشكەن نەرسە، كۆزنەكنى تاقىسىڭىزمۇ كەسىپ جەريانى ئۆچمەيدۇ |
| **ھەقىقىي ۋاقىت Timeline** | باش بەت SSE ئارقىلىق بارلىق AI ↔ ئىشلەتكۈچى ئۇچۇرلىرىنى ھەقىقىي ۋاقىتتا ئىتتىرىدۇ، TransitionGroup ئانىماتسىيەسى |
| **TOTP قوش فاكتورلۇق دەلىللەش** | ئىچكى 2FA، ئىشەنچلىك ئۈسكۈنە بارماق ئىزى（FingerprintJS）+ Cookie Token قوش يولنى قوللايدۇ |
| **ئودىت خاتىرىسى** | بارلىق سەزگۈر مەشغۇلاتلار（باغلاش، كىرىش، Terminal غا ئۇلاش）ئودىت خاتىرىسىگە يېزىلىدۇ |
| **macOS ئۇسلۇبىدىكى تېما** | Tailwind + shadcn/ui قارا-ئاق-كۈلرەڭ ئۇسلۇبى، Light/Dark ئالماشتۇرۇشنى قوللايدۇ |
| **PM2 بىر كۇنۇپكا بىلەن ئورۇنلاشتۇرۇش** | ئۆز ئىچىگە ئالغان `deploy.sh` + `ecosystem.config.cjs`، مەنبە كودى بىلەن ئىجرا مۇھىتى ئايرىلغان |

```

│  ئىشلەتكۈچى       │ ───────────────────▶│  飞书 ئوچۇق سۇپىسى   │

                              ┌─────────────────────────┐

                              │ (Express + Vue + WS)    │

                              ┌─────────────────────────┐
                              │  يەرلىك ياكى يىراق ماشىنا          │
                              │  ┌─────┐  ┌─────┐       │
```

تېخنىكا قۇرۇلمىسى：
- **ئالدى يۈزى**：Vue 3 + Vite + TypeScript + Pinia + Tailwind + xterm.js
- **ئارقا يۈزى**：Node.js 20+ + Express + ws + node-pty + ssh2
- **ساندان**：SQLite + Drizzle ORM
- **جەريان باشقۇرۇش**：tmux + PM2

## 🚀 تېز باشلاش

### 1. مۇھىت تەلىپى

- Node.js ≥ 20
- npm ≥ 10（ياكى pnpm）
- tmux ≥ 3.0（CLI جەريانى باشقۇرۇش）
- كەم دېگەندە بىر AI CLI ئىجرا قىلىشقا بولىدىغان بولۇش：`claude`（Claude Code）ياكى `codex`
- بىر 飞书 كارخانا ئۆزى قۇرغان ئەپ（`im:message`、`im:message.group_at_msg` نى ئېچىش كېرەك）

### 2. يەرلىكتە قوزغىتىش

```bash
# كلونلاش
git clone https://github.com/LengendXing/feishu-claudecode-bridge.git
cd feishu-claudecode-bridge

# بېقىندىلىقلارنى قاچىلاش
npm install

# مۇھىت ئۆزگەرگۈچىلىرىنى كۆچۈرۈش
cp .env.example .env
# .env نى تەھرىرلەپ، JWT_SECRET قاتارلىقلارنى تولدۇرۇڭ

# سانداننى دەسلەپلەشتۈرۈش + ئۇرۇق ھېساباتى
npm run db:migrate
npm run seed

# تەرەققىيات ھالىتى（ئالدى-ئارقا يۈزى تەڭ ئىجرا）
npm run dev
```

`http://localhost:3456/` نى زىيارەت قىلىپ، سۈكۈتتىكى ھېسابات `nimasile` / `.env` دا تەڭشەلگەن `ADMIN_PASSWORD` بىلەن كىرىڭ.

## 📦 ئورۇنلاشتۇرۇش كۆرسەتمىسى

### ئۇسۇل 1：بىر كۇنۇپكا بىلەن يىراق ئورۇنلاشتۇرۇش（تەۋسىيە）

نىشان مۇلازىمېتىردا ئىجرا قىلىڭ：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/LengendXing/feishu-claudecode-bridge/main/cll.sh)
```

قاچىلاش مۇندەرىجىسىنى بەلگىلەش：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/LengendXing/feishu-claudecode-bridge/main/cll.sh) /opt/feishu-bridge
```

ئورۇنلاشتۇرغاندىن كېيىنكى مۇندەرىجە قۇرۇلمىسى：

```
feishu-claudecode-bridge/
├── sourceCode/   ← مەنبە كودى（git pull بىلەن يېڭىلاش）
└── deploy/       ← ئىجرا مۇھىتى（PM2 بۇ يەردىن قوزغىلىدۇ）
```

### ئۇسۇل 2：قوللىنىپ PM2 ئورۇنلاشتۇرۇش

```bash
git clone https://github.com/LengendXing/feishu-claudecode-bridge.git
cd feishu-claudecode-bridge
bash deploy.sh
```

`deploy.sh` تۆۋەندىكىلەرنى قىلىدۇ：
1. `npm ci` بېقىندىلىقلارنى قاچىلاش
2. `npm run build` ئالدى-ئارقا يۈزىنى قۇرۇش
3. `dist/`、`scripts/`、`package.json` قاتارلىقلارنى `../deploy/` غا كۆچۈرۈش
4. `../deploy/` دا `pm2 start ecosystem.config.cjs` ئىجرا قىلىش

### ئۇسۇل 3：كۈندىلىك يېڭىلاش جەريانى

```bash
cd sourceCode/
git pull
bash deploy.sh   # ئاپتوماتىك قۇرۇش + PM2 نى قايتا قوزغىتىش
```

### تۇنجى قېتىم ئورۇنلاشتۇرغاندىن كېيىن قىلىشقا تېگىشلىك ئىشلار

1. `deploy/.env` نى تەھرىرلەپ، `JWT_SECRET`、`ADMIN_PASSWORD` نى تولدۇرۇڭ
2. بروۋزېردا `http://<مۇلازىمېتىر IP>:3456/` نى زىيارەت قىلىڭ
3. سۈكۈتتىكى ھېسابات بىلەن كىرگەندىن كېيىن، «تەڭشەك» كە كىرىپ 2FA نى ئېچىڭ（كۈچلۈك تەۋسىيە）
4. «ماشىنا» مېيۇسى：سۈكۈتتە يەرلىك ماشىنا تەييار؛ يىراق ماشىنا كېرەك بولسا، يېڭىدىن قۇرۇپ SSH ئىسپاتىنى تولدۇرۇڭ
5. «مۇلازىمەت بىلەن تەمىنلىگۈچى» مېيۇسى：يېڭىدىن Anthropic / OpenAI / ئۆزى بەلگىلەنگەن مۇلازىمەت بىلەن تەمىنلىگۈچى قۇرۇپ، base_url + API Key تولدۇرۇڭ
6. مۇلازىمېتىردا tmux session قوزغىتىڭ：`tmux new-session -d -s cc-work`
7. «باغلاش» مېيۇسى：يېڭى 飞书 بوتى باغلاش قوشۇپ، App ID / App Secret / Verification Token / Encrypt Key + CLI تاللاش + مۇلازىمەت بىلەن تەمىنلىگۈچى + مودېل + Effort تولدۇرۇڭ
8. 飞书 توپىدا بوتنى @ قىلىپ خالىغان ئۇچۇر ئەۋەتسىڭىز → ئارقا سۇپا ئاپتوماتىك cc/codex جەريانىنى قوزغىتىپ كۆۋرۈكلەشتۈرىدۇ

## 📘 ئىشلىتىش ھۆججىتى

### 飞书 ئەپ تەڭشىكى

1. [飞书 ئوچۇق سۇپىسى](https://open.feishu.cn/) غا كىرىپ، **كارخانا ئۆزى قۇرغان ئەپ** قۇرۇڭ
2. ھوقۇق ئېچىش：
   - `im:message`（ئۇچۇر ئەۋەتىش）
   - `im:message:receive_v1`（ئۇچۇر قوبۇللاش）
   - `im:message.group_at_msg`（توپ ئىچىدە @ قوبۇللاش）
3. ۋەقەلىك مۇشتەرى بولۇش：**ئۇزۇن ئۇلىنىش ھالىتى**（تەۋسىيە）نى ئېچىڭ ياكى Request URL `https://<host>/webhook/feishu` نى تەڭشەڭ
4. App ID / App Secret / Verification Token / Encrypt Key نى كۆچۈرۈپ → Bridge نىڭ «باغلاش» تەڭشىكىگە چاپلاڭ

### باغلاش قۇرۇش

«باغلاش» مېيۇسىغا كىرىڭ → يېڭىدىن قوشۇڭ：
- **باغلاش نامى**：ئۆزى بەلگىلەش（مەسىلەن `cc-projectA`）
- **ماشىنا（host）**：يەرلىك ياكى ئالدىن قۇرۇلغان يىراق ماشىنىنى تاللاش
- **CLI تىپى**：cc / codex
- **مۇلازىمەت بىلەن تەمىنلىگۈچى**：قۇرۇلغان مۇلازىمەت بىلەن تەمىنلىگۈچىنى تاللاش
- **مودېل**：مۇلازىمەت بىلەن تەمىنلىگۈچىدىن ئاپتوماتىك تەكشۈرۈلىدۇ؛ تەكشۈرۈش مەغلۇپ بولسا سۈكۈتتىكى مودېلنى قولدا تاللاش ياكى ئۆزى بەلگىلەنگەن مودېل ID نى كىرگۈزۈشكە بولىدۇ
- **Effort**：مودېل maxEffort بويىچە تاللاشقا بولىدىغان دەرىجىلەرنى كۆرسىتىدۇ
- **飞书 ئەپ تۆت پارچىسى**：App ID / Secret / Verification Token / Encrypt Key

ساقلىغاندىن كېيىن، Bridge ئاپتوماتىك 飞书 ئۇزۇن ئۇلىنىش WS غا ئۇلىنىشنى سىنايدۇ، ھالەت `online` غا ئايلانغاندىن كېيىن 飞书 توپىدا بوتنى @ قىلىپ ئىشلەتسىڭىز بولىدۇ.

### Web Terminal

«باغلاش» تىزىملىكىدە `Terminal` كۇنۇپكىسىنى بېسىپ، بروۋزېردا بىۋاسىتە xterm ئاچىسىز، بۇ يەرلىكتە ئىجرا قىلغانغا باراۋەر：

```bash
tmux attach -t cc-projectA
```

**مۇھىم بىخەتەرلىك چەكلىمىسى：بروۋزېر Terminal Tab نى تاقىسىڭىزمۇ tmux session ئۆچمەيدۇ، كەسىپ جەريانى داۋاملىق ئىجرا بولىدۇ**.

Ctrl-b d بىلەن ئاكتىپلىق بىلەن detach قىلىشقا بولىدۇ، ResizeObserver كۆزنەك چوڭلۇقىنى ماس قەدەمدە يېڭىلايدۇ، 5 مىنۇت مەشغۇلات بولمىسا WebSocket ئاپتوماتىك ئۈزۈلىدۇ（كەسىپ جەريانىغا تەسىر كۆرسەتمەيدۇ）.

### ھەقىقىي ۋاقىت Timeline

باش بەت ئاستى Timeline رايونى يېقىنقى 20 دانە 飞书 ئۇچۇرىنى كۆرسىتىدۇ（SSE ھەقىقىي ۋاقىتتا ئىتتىرىلىدۇ），يېڭى تۈرلەر ئۈستىدىن scale+fade سىيرىلىپ كىرىدۇ، چېكىلسە تولۇق مەزمۇننى يايىدۇ/يىغىدۇ، سۇپا tag رەڭ بويىچە پەرقلىنىدۇ（飞书 يېشىل / Telegram كۆك زاپاس）.

### سىستېما خاتىرىسى

«خاتىرە» مېيۇسى：SSE ئارقىلىق ئارقا يۈز ئىجرا خاتىرىسىنى ھەقىقىي ۋاقىتتا دومىلىتىپ كۆرسىتىدۇ（يېقىنقى N قۇرنى قايتا قويۇش + يۈرەك سوقۇش ساقلاشنى قوللايدۇ）.

### تېما ئالماشتۇرۇش

ئوڭ تەرەپ ئۈستۈنكى بۇرجىكىدىكى قۇياش/ئاي سىنبەلگىسى بىلەن Light / Dark ھالىتىنى ئالماشتۇرىدۇ، تېما رەڭگى قارا-ئاق-كۈلرەڭ سىستېمىسىدا.

## 📱 ماس كېلىدىغان ئاندروئىد تېرمىنالى bacs-android

> پروژېكت ئادرېسى：[https://github.com/LengendXing/bacs-android](https://github.com/LengendXing/bacs-android)

**bacs-android** بولسا بۇ سىستېمىنىڭ رەسمىي ئاندروئىد تېرمىنال ئەپى، يانفوندا تۆۋەندىكىلەرنى قىلالايسىز：

- 🔔 Bridge Timeline ئىتتىرىشىنى ھەقىقىي ۋاقىتتا قوبۇللاش، AI جاۋابى IM ئۇچۇرىغا ئوخشاش يېتىپ كېلىدۇ
- ⌨️ يانفوندا بىۋاسىتە بۇيرۇق كىرگۈزۈپ Bridge غا قايتۇرۇپ ئەۋەتىش، 飞书 نى ئېچىش ھاجەتسىز
- 📊 بارلىق باغلانغان CLI جەريانلىرىنىڭ ھالىتىنى كۆرۈش（online / offline / awaiting_choice）
- 📜 تارىخىي سېشن ۋە سىستېما خاتىرىسىنى كۆرۈش
- 🔐 TOTP قوش فاكتورلۇق كىرىش + ئۈسكۈنە بارماق ئىزى ئىشەنچى
- 🌙 سىستېمىغا ئەگىشىدىغان Light / Dark تېما

**ئۇلىنىش ئۇسۇلى**：App نى قوزغاتقاندىن كېيىن Bridge Server ئادرېسىنى（مەسىلەن `http://192.168.1.100:3456`）كىرگۈزۈپ، تەڭشەك بېتىدىكى تېز كىرىش ئىككىلىك كودىنى سىكاننېرلىسىڭىز بىر كۇنۇپكا بىلەن كىرىش تاماملىنىدۇ（60s قىسقا مۇددەتلىك JWT → كلىيېنت ئۇزۇن مۇددەتلىك token ئالماشتۇرۇش لايىھەسى، ئىككىلىك كود ئاشكارىلىنىپ قېلىشتىن ساقلىنىدۇ）.

**ئاندروئىد تېرمىنالى** تولۇق ھۆججىتىنى [bacs-android README](https://github.com/LengendXing/bacs-android#readme) دىن كۆرۈڭ.

## ❓ دائىم سورالغان سوئاللار

**س：飞书 ئۇچۇر ئەۋەتكەندىن كېيىن جاۋاب يوقمۇ؟**
جا：ئالدى بىلەن تەكشۈرۈڭ：① باغلاش ھالىتى `online` مۇ؛② مۇلازىمېتىر tmux session ساقلانغانمۇ؛③ مۇلازىمەت بىلەن تەمىنلىگۈچى API Key ئۈنۈملۈكمۇ؛④ «خاتىرە» مېيۇسىدە ئارقا يۈز خاتىرىسىنى ھەقىقىي ۋاقىتتا كۆرۈڭ.

**س：يىراق ماشىنىغا كىرىش مەغلۇپ بولدى（Not logged in）？**
جا：v1.0.8 دە ئوڭشالدى——`bash -ilc` بىلەن ئورالغان tmux قوزغىتىش بۇيرۇقىنى ئىشلىتىپ يىراق rc ھۆججىتىنى يۈكلەشكە كاپالەتلىك قىلىڭ. يەنىلا مەسىلە بولسا، يىراق `~/.bashrc` تا `claude` / `codex` ئىجرا قىلىشقا بولىدىغان-بولمايدىغانلىقىنى تەكشۈرۈڭ.

**س：SSH ئۇزۇن ۋەزىپە مەزگىلىدە ئالاقە ئۈزۈلدىمۇ؟**
جا：v1.0.10 دە ئوڭشالدى——يۈرەك سوقۇش 5 مىنۇتتىن 30s كە ئۆزگەرتىلدى، ھەمدە 60s لىق ئاكتىپ بىكارلىق ئۈزۈش چىقىرىۋېتىلدى.

**س：tmux session مەۋجۇت بولمىسا قانداق بىر تەرەپ قىلىنىدۇ؟**
جا：Bridge «باغلاش سىزىققا چىقىش» تا مۇناسىپ session نى（مەسىلەن `cc-xxx`、`codex-xxx`）ئاپتوماتىك قۇرىدۇ، قولدا مەشغۇلات قىلىش ھاجەتسىز.

**س：Web Terminal بروۋزېر كۆزنىكىنى تاقىسا كەسىپ جەريانى ئۆچەمدۇ؟**
جا：**ئۆچمەيدۇ**. Web Terminal پەقەت tmux كلىيېنتىنى detach قىلىدۇ، كەسىپ session داۋاملىق ئىجرا بولىدۇ. كود قاتلىمىدا مەجبۇرىي مۇداپىئە بار：`session-name` ئاق تىزىملىك مۇنتىزىمىدىن ئۆتىدۇ، تاقاش يولى ھەرگىز `tmux kill-session` نى چاقىرمايدۇ.
